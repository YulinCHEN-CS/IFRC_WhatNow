research

similar project

The IFRC Alert Hub (UCL Project) is a project that involves the use of technology to improve the efficiency of disaster response. The project showcased the development and implementation of an alarm system based on the latest technology through several GitHub repositories. It mainly includes the development of front-end interface, the construction of back-end services, the aggregation and processing of data, and the management of user subscription systems.

By studying this project in depth, we gained knowledge on how to design and implement a responsive front-end interface that can effectively present alert information to end users. TypeScript used in the project provides the advantages of strong typing and modern JavaScript features, making the development process more efficient and reliable.

On the backend, the project developed a CAP aggregator and alert manager using Python to demonstrate how to process and aggregate alert data from different sources. It highlights the importance of data processing and how to ensure the accuracy and reliability of the data.

In addition, the subscription system shows how to manage user subscription information and how to push relevant alert messages based on user preferences. The privacy of user data is also something that our project can learn from.

resource: https://github.com/IFRC-Alert-Hub/

Technology Review

Node.js and Python each have their own advantages and disadvantages in backend development, and our team mainly chose between these two tools

Node.js

Speed and Performance: Node.js executes code outside of the web browser, which allows for efficient handling of concurrent requests and high-performance outcomes. This makes it ideal for real-time applications (RTAs), such as chat apps and live updates.[1]

Scalability: With features like the Cluster Module, Node.js enables easy scaling of applications across multiple CPUs, making it well-suited for applications that anticipate growth in user numbers or data processing needs.[1]

Unified JavaScript Development: Node.js allows developers to use JavaScript for both client-side and server-side scripting. This unification can streamline development processes, potentially reducing development time and effort[2]

python

Simplicity and Readability: Python's syntax is designed for clarity and simplicity, making it an excellent choice for beginners and projects that prioritize maintainability and readability[3]

Rich Ecosystem: Python boasts a rich ecosystem of libraries and frameworks (e.g., Django, Flask) that can accelerate the development of a wide variety of applications

conclusion

finally, We chose Node.js, not only because of the above advantages, but also because most of our team members found it more convenient to use Node.js

Tabular Data Extraction: Explore & Solve

Background

One of the main tasks in our research was to extract tabular data from PDF files. The diversity and complexity of PDF formats make this task challenging. We tried a variety of tools and methods with the goal of finding a solution that was both accurate and efficient.

First attempt

Our first attempt was to utilize pdf-table-extractor. This tool works well when working with a single column of data, but not so well when encountering multiple columns of data. It tends to merge all the data into one column, resulting in loss of data structure. This obviously doesn't meet our need for accuracy.

Next, we turned to tablula.js. tablula.js is more flexible when it comes to handling tabular data, but one of its main problems is that the range of recognition is too wide. It often incorrectly generat non-tabular content, such as headers, footers, and surrounding text, as tabular data, which adds to the difficulty and complexity of subsequent cleanup efforts.

Further explore

In order to solve these problems more perfectly, we tried to find a new package, pdf2table. Although this tool performs relatively well when dealing with certain types of PDF forms, overall, it lacks recognition accuracy. After several trials, we found that it could not reliably handle complex table layouts or non-standard table formats.

Final solution

After thorough research and testing, we finally decided to go with pdf2json. The key advantage of this tool is that it can extract the exact coordinates of the text in the PDF. Using this coordinate information, we can accurately reconstruct the rows and columns of the table, ensuring that the structure and integrity of the data are maintained. We successfully reconstructed the table by writing a specific algorithm to parse this coordinate data. This provides us with a reliable way to process tabular data in PDF files.
