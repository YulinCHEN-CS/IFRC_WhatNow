Critical evaluation

user experience

·The design of the WHATNOW website provides a clean and distraction-free user interface, which is particularly important in emergency situations. Users can quickly locate the information and features they need, such as harzard content management, Audit logs, bulk uploads, and WhatNow message. These elements are presented through clear labels and intuitive icons, further simplifying the decision-making and action process for users.

·Clear and conspicuous buttons will help users to go to the next action or go back to the previous step.

Functionalities

The site has built a logical, intuitive ADMIN working area, allowing users to navigate to the sections they require in an intuitive manner. The bulk upload feature is a standout example, enabling rapid content updates at critical moments, which is vital in emergency and rapidly changing situations. This functionality reduces the time consumed in individual data entry, enhancing response speeds.

The content management system and audit log of the WHATNOW website are designed with user efficiency and accountability in mind. The intuitive content management interface allows administrators to manage disaster information effectively, reducing errors and ensuring the accuracy and timeliness of critical data. Simultaneously, the comprehensive audit log system meticulously tracks all user activities, offering invaluable insights for both post-event auditing and real-time monitoring, t hereby enhancing the website's transparency and reinforcing accountability measures.

Stability

Robustness of the system: WHATNOW has an important responsibility for disaster warning and response measures, therefore, the system must be able to handle high traffic and data loads, especially in emergency situations. Our robust back-end architecture maintains service continuity and stability when multiple users access at the same time.

Data consistency: For content management systems, ensuring data consistency is critical. When the administrator adds, deletes, modifies, and checks, the corresponding database will be synchronized, and all changes should be reflected in real time and synchronized to all user interfaces correctly.

Efficiency

Data processing speed: The Bulk Upload feature can efficiently process large amounts of data input. The CSV file upload and parsing process is fast and does not have a significant impact on system performance.

User interface responsiveness: The front-end design allows users and administrators to quickly and intuitively access information and functionality, whether on desktop or mobile devices. Load times are kept to a minimum, especially on data-intensive pages.

Resource utilization: The efficient use of server resources is essential to maintain system efficiency. What we have done in this area needs to be improved. In the case of processing large datasets and high user concurrency, resource usage cannot be optimized.

Compatibility

We have deployed our website on the Azure platform, which has greatly improved the compatibility of our website.

1. Mobile Device Compatibility: The website provides native apps on mobile platforms and ensures that the user experience is consistent across screen sizes and operating systems.

2. Global Accessibility: The website is accessible worldwide.

3. Browser Compatibility: The front-end design of the WHATNOW website is optimized using the tools and services provided by Azure and performs well on all major browsers, including Chrome, Firefox, Safari, Edge, etc

Maintainability

Error handling: The Audit Log system will record all user activities, which will make error handling easier and faster, and will make it easier to locate and handle errors.

Advanced Tools: The project was developed using MySQL, Node.js Express, React, and Python to improve the maintainability of the project

Project management

The project is well managed. Each member is assigned a different job. And schedule deadlines for each part of the project. We also meet with our clients regularly and communicate within the group on a weekly basis.

Future work

Further Work for WhatNow Website:

Responsive Layout Implementation:

One of the key areas for further enhancement is to implement a responsive layout for the WhatNow website. This involves redesigning the website's layout and interface to ensure optimal viewing and usability across different devices and screen sizes. By utilizing responsive design techniques such as fluid grids, flexible images, and media queries, we can create a seamless browsing experience for users accessing the website from desktops, laptops, tablets, and smartphones.

Multilingual Translation Integration:

Another important aspect to consider is the integration of multilingual translation capabilities into the WhatNow website. This involves translating the website's content, including text, labels, and messages, into multiple languages to cater to a diverse global audience. By providing multilingual support, we can enhance accessibility and reach a broader user base, ultimately improving user satisfaction and engagement.

User Acceptance Test (UAT) Driven Optimization:

Conducting thorough user acceptance tests (UAT) is crucial for identifying areas of improvement in both the user interface (UI) and features of the WhatNow website. Based on the feedback and insights gathered from UAT sessions, we can prioritize and implement optimizations to enhance the overall user experience. This may include refining navigation flows, improving usability, addressing usability issues, and adding or enhancing features based on user preferences and expectations.

Performance Optimization and Scalability:

Further work also involves optimizing the performance and scalability of the WhatNow website to ensure smooth and efficient operation, especially during peak usage periods or when handling a large volume of traffic. This may involve optimizing code, minimizing load times, implementing caching mechanisms, and scaling infrastructure resources as needed to accommodate growing user demands.

Accessibility Enhancement:

Improving accessibility is another important area for further work. This involves ensuring that the WhatNow website is accessible to users with disabilities, including those using assistive technologies such as screen readers or voice recognition software. By adhering to accessibility standards and guidelines, we can ensure that all users can access and interact with the website effectively, regardless of their abilities or impairments.

In conclusion, by focusing on responsive layout implementation, multilingual translation integration, UAT-driven optimization, performance optimization and scalability, and accessibility enhancement, we can further enhance the WhatNow website to better meet the needs and preferences of its users, ultimately improving user satisfaction and engagement.
